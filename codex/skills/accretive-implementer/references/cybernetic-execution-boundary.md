# Cybernetic Execution Boundary

`$accretive-implementer` should not independently perform broad system diagnosis.

If a `cybernetic_context` is supplied, preserve it.

## Required behavior

- Implement only the selected downstream route.
- Preserve forbidden local optimizations.
- Do not turn a feedback/rule/goal intervention into a local parameter tweak.
- If implementation discovers the selected route is insufficient, stop and return to the route owner.

## Reject

- raw local patch request that conflicts with `local_patch_allowed: no`;
- add-new-surface without system/rent/route warrant;
- metric/proxy change without incentive scan.
