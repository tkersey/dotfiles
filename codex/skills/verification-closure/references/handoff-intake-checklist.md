# Handoff intake checklist

Use this checklist before trusting a Closure Handoff Packet.

- Are all required headings present and in order?
- Is `Artifact State Label` current?
- Do any specialist briefings have mismatched state labels?
- Does the packet still show every previously open material issue?
- Do status transitions cite evidence?
- Does the verification ledger distinguish:
  - direct changed path
  - claimed failure mechanism
  - regression surface
- Do the requested closure questions match the actual open gates?
- Is any field missing, silently merged, or replaced with vague prose?

If any answer is no, mark the handoff contract `incomplete` or `stale`.
