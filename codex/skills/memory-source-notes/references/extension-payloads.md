# Extension Payload Contracts

## Harness Rule

```json
{
  "harness_rule": "When X, do Y before Z.",
  "trigger": "Situation or symptom",
  "preferred_behavior": "Future behavior",
  "failure_avoided": "Predictable failure",
  "verification_cue": "Proof or stop condition",
  "evidence_count": 1,
  "repetition_count": 1
}
```

Required for `harness-rule`: all fields except repetition count.

## Learning Admission

```json
{
  "learning_id": "lrn-...",
  "learning_status": "codify_now",
  "repo": "owner/repo",
  "source_path": ".learnings.jsonl",
  "decision_delta": "What future Codex should do differently",
  "evidence_snapshot": ["Exact evidence anchor"],
  "future_behavior": "Default, route, or procedure",
  "verification": "Proof or stop condition",
  "tags": ["memory-admission"]
}
```

The complete canonical row remains in `.learnings.jsonl`.

## Negative-Ledger Projection

```json
{
  "neg_id": "NEG-000001",
  "record_version": "NER-v2",
  "ledger_path": ".ledger/negative-ledger.jsonl",
  "projection_fingerprint": "...",
  "status": "active",
  "kind": "realization_route",
  "route_or_model_id": "same-leaf-batching",
  "cluster_id": "mvcc-small-write",
  "artifact_state_id": "HEAD",
  "hypothesis": "...",
  "attempted_change": "...",
  "observed_outcome": "...",
  "failure_class": "local-regression",
  "source_refs": [],
  "falsifying_evidence": [],
  "exclusion_scope": "route_family",
  "exclusion_rule": "...",
  "applicability_conditions": [],
  "reopening_criteria": [],
  "confidence": "high",
  "next_search_hint": "..."
}
```

An `active` projection is admissible only when generated by `ledger export` from a valid ledger record.

## Synesthesia Mapping

```json
{
  "sensory_phrase": "long corridor",
  "engineering_translation": "serialized waits or amplified latency",
  "activation_boundary": "performance or dependency-chain diagnosis",
  "non_activation_boundary": "exact syntax, legal, compliance, security sign-off",
  "scope": "task_family_scoped",
  "scope_anchor": "performance-triage",
  "endorsement_type": "explicit-user-endorsement",
  "verification": "Every use translates back into concrete engineering implications"
}
```

A mapping requires explicit endorsement/correction/rejection or repeated accepted operational use.
