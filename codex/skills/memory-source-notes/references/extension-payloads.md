```json
{
  "learning_id": "lrn-...",
  "learning_status": "codify_now",
  "repo": "owner/repo",
  "source_path": ".ledger/learnings/events.jsonl",
  "decision_delta": "What future Codex should do differently",
  "evidence_snapshot": ["Exact evidence anchor"],

```

## Synesthesia mapping

The canonical envelope owns authority, scope, source evidence, and prior-note relationships. The skill-facing payload owns only the reversible sensory mapping:

```json
{
  "sensory_phrase": "long corridor",
  "engineering_translation": "serialized waits or amplified dependency latency",
  "activation_boundary": "performance or dependency-chain diagnosis",
  "non_activation_boundary": "exact syntax or literal-only requests",
  "verification": "Every use names the concrete wait or latency mechanism and evidence"
}
```

For a rejection, replace `engineering_translation` with:

```json
{
  "rejection_reason": "Why this phrase should no longer guide diagnosis"
}
```

The Synesthesia adapter generates these compatibility fields for the current native writer:

```text
payload.scope
payload.scope_anchor
payload.endorsement_type
```

They are derived from the envelope and are not independent authorities. A caller may supply them only when they exactly match the generated values.

A mapping requires explicit endorsement, correction, rejection, retraction, reopening, or repeated accepted operational use. Explicit durable user authority is sufficient without repetition. Without explicit durability, require accepted use across at least two independent contexts.

## Synesthesia activation boundary

```json
{
  "activation_boundary": "Use for explicit compare-by-feel requests",
  "non_activation_boundary": "Do not activate for ordinary architecture or performance work",
  "verification": "Future activation names explicit sensory language or a documented representational ambiguity"
}
```

## Synesthesia boundary retraction

```json
{
  "retracted_boundary": "Implicit use for every performance task",
  "reason": "Performance alone does not establish representational need",
  "verification": "Future activation requires explicit sensory language or a documented handoff"
}
```

## Synesthesia generated digest

A successful validated Synesthesia append refreshes this non-canonical materialized view:

```text
${CODEX_HOME:-$HOME/.codex}/memories/extensions/synesthesia/resources/latest_synesthesia_digest.md
```

The digest folds immutable operation chains into active mappings and boundaries. It must retain `source_note_ids`, a current source fingerprint, unresolved-chain diagnostics, and inactive states. It never becomes independent admission authority.
