# Extension Payload Contracts

## Harness Rule

```json
{
  "harness_rule": "When X, do Y before Z.",
  "trigger": "Situation or symptom",
  "preferred_behavior": "Future behavior",
  "failure_avoided": "Predictable failure",
  "verification_cue": "Proof or stop condition",
  "evidence_count": 1,
  "repetition_count": 1
}
```

Required for `harness-rule`: all fields except repetition count.

## Learning Admission

```json
{
  "learning_id": "lrn-...",
  "learning_status": "codify_now",
  "repo": "owner/repo",
  "source_path": ".learnings.jsonl",
  "decision_delta": "What future Codex should do differently",
  "evidence_snapshot": ["Exact evidence anchor"],
  "future_behavior": "Default, route, or procedure",
  "verification": "Proof or stop condition",
  "tags": ["memory-admission"]
}
```

The complete canonical row remains in `.learnings.jsonl`.

## Synesthesia Mapping

The source-note envelope, not the payload, owns operation, authority, scope, source references, related IDs, and supersession.

```json
{
  "sensory_phrase": "long corridor",
  "engineering_translation": "serialized waits or amplified dependency latency",
  "activation_boundary": "performance or dependency-chain diagnosis",
  "non_activation_boundary": "literal-only, exact syntax, legal/compliance, or security sign-off",
  "verification": "Every use identifies the concrete wait or latency mechanism and evidence"
}
```

Allowed payload fields:

```text
sensory_phrase
engineering_translation
activation_boundary
non_activation_boundary
verification
```

Do not put these envelope-owned fields in the payload:

```text
operation
authority
scope
scope_anchor
endorsement_type
source_refs
related_ids
supersedes_id
```

A mapping requires explicit endorsement, correction, rejection, retraction, direct remember request, or independently evidenced repeated accepted operational use.

Before `memory-note append`, the owning Synesthesia skill must validate and canonicalize the entire input envelope:

```bash
uv run --with pyyaml python \
  codex/skills/synesthesia/scripts/validate_synesthesia.py \
  memory-file proposed.json --kind <synesthesia-kind> --emit-canonical \
  > proposed.canonical.json
```

`$memory-source-notes` must pass those canonical bytes to `memory-note` unchanged. This gives semantically equivalent Synesthesia inputs stable byte representation for duplicate detection.
