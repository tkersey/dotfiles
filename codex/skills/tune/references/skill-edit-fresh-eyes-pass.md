# Skill-Edit Fresh-Eyes Pass for Tune

For `apply-with-refine`, before the final report, reread the modified skill as both a user and the router.

Check:
- Did the trigger description become too broad or too narrow?
- Did the edit conflict with companion-skill boundaries?
- Did the workflow absorb work owned by another skill?
- Did the change weaken evidence, privacy, publishing, or handoff requirements?
- Did it introduce naming drift, stale paths, duplicate sections, frontmatter violations, or false activation risk?
- Would the edit create missed activations, false activations, or partial activations relative to the diagnosed gap?

If the pass finds a material issue, fix it before the final report. If it finds no material delta, record `fresh_eyes_delta: none` under Outcome observation or Remaining uncertainty.
