# Using EPG Evidence in `$tune`

Use transition evidence to tune skills and policies:

```text
predicted vs observed effects
critical unknown resolution latency
same-state/action recurrence
shield intervention
model and intent failures
surface/proof prediction error
rollback and discarded work
terminal regret candidates
```

Strong evidence:

```text
same evidence state
controlled alternative or replay
stable artifact lineage
route/prediction delta
```

Weak evidence:

```text
policy mention
skill presence
successful final outcome alone
self-report without ETR/receipt
```

Tune the smallest responsible layer:

```text
source/spec
policy compiler
runtime selector
st materializer
action realizer
proof model
negative-memory route
```
