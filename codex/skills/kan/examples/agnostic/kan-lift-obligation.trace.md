# Kan lift obligation trace

## Problem

A test suite describes accepted public behavior, but the architecture needs derived internal obligations that are sufficient and not ad hoc.

## Lift data

- `A`: test/spec cases.
- `B`: internal obligations or capability sets.
- `C`: observable behavior.
- `P : B -> C`: behavior exposed by satisfying obligations.
- `F : A -> C`: accepted behavior described by tests.
- Candidate: `Rft_P F`.
- Comparison: `ε : P · Rft_P F -> F`.

## Witness

- `a`: idempotent retry test for payment capture.
- `F(a)`: accepted observable traces have one successful charge.
- `Rft_P F(a)`: obligation set `{idempotency-key uniqueness, atomic charge recording}`.
- `P(Rft_P F(a))`: traces generated by implementations satisfying those obligations.

## Law test

```text
P(Rft_P F(a)) <= F(a)
```

Concrete regression:

```text
all traces generated by the obligation harness satisfy the retry test oracle
```

## Failure mode prevented

Tests remain behavioral examples only and never become architecture-level obligations, so future implementations pass local tests while violating global idempotency.
