# Anti-patterns

- Calling a repository layout a category without identifying objects, morphisms, and preservation.
- Promising codensity speedups without semantic tests and benchmarks.
- Letting plugin authors reimplement core semantics instead of factoring through a central `Lan`/interpreter.
- Building compatibility facades one endpoint at a time without `Ran`-style coherence tests.
- Treating contract tests as mere pass/fail instead of lift inputs that derive obligations.
- Claiming Freyd/AFT applies to a projection without constraints, templates, and preservation tests.
