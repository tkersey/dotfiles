# Cybernetic Failure Patterns

Use `$cybernetic` with `$negative-ledger` when failed routes form a system pattern.

## Add to negative evidence capture

```yaml
cybernetic_failure_pattern:
  system_pattern: "..."
  feedback_loop: "..."
  incentive_or_proxy_failure: yes | no
  delayed_feedback: yes | no
  local_optimization_harmed_whole: yes | no
  repeated_route_family: "..."
  next_leverage_hint:
    route: use_information_flow | change_rule | change_goal | use_distillation | use_universalist | use_reduce | blocked
```

## Rule

Negative-ledger says what failed. Cybernetic explains what system structure made that failed route attractive but ineffective.
