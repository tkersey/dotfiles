# Negative Evidence Integration

A repeated review decision is a falsified hypothesis.

When the same cluster reappears after a selected route, record the selected route as negative evidence unless it is proven stale, unrelated, or superseded.

The next packet must either:

1. avoid the active excluded route;
2. prove the negative evidence stale/superseded/reopened;
3. or block.

Negative evidence must be present when:

- same cluster reappears after repair;
- selected normal form was falsified;
- universalist not-needed was falsified;
- add-surface route failed or became unsound;
- public bypass / compatibility / tolerance path was introduced and then rejected;
- Review Distillation Mode is active.
