# Common CLI reporting

Assume the user may only see the last screenful of terminal output.

## Tail-weighting rules
- Keep dense evidence and ledgers above.
- Put the decisive action, verdict, or gate at the bottom.
- Make the final section self-sufficient when read alone.
- Never bury the exact next move only inside rationale prose.

## Good final sections
- implementation: `Execution Bottom Line`
- review: `Change Agenda` then `Reviewer Bottom Line`
- orchestration: `Final State` then `Do Next`
- closure: `Reopen Trigger` then `Closure Bottom Line`
- adjudication: `Handoff Agenda` then `Adjudication Bottom Line`

## Compression rule
Use terse early sections and richer bottom-line sections.
If the report is long, compress supporting evidence before compressing the final action.
