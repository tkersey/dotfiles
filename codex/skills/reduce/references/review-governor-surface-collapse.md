# Review Governor Surface Collapse

After same-cluster recurrence, `$reduce` should be considered when code growth is validation/evidence/predicate accretion.

Use `$reduce` to ask:

- can duplicated predicates collapse?
- can branches become an existing table/model?
- can a compatibility path be deleted by moving the boundary?
- can helper/wrapper/adapter surface be retired?
- can tests be merged into a proof matrix?

If yes, prefer delete-collapse-canonicalize over positive production growth.
