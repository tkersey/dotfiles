# Cybernetic Abstraction Tax

Use `$cybernetic` before deleting an abstraction that appears in a repeated system pattern.

## Classify the surface

```yaml
abstraction_system_role:
  surface: "..."
  role: layer_tax | feedback_channel | boundary_artifact | buffer | bottleneck | control_point | proxy_metric | unknown
  cybernetic_context_ref: "..."
```

## Route

- `layer_tax` -> `$reduce`
- `boundary_artifact` -> `$universalist`
- `feedback_channel` / `control_point` -> preserve or redesign
- `proxy_metric` -> redesign measurement/incentive
- `buffer` -> tune capacity, delay, or flow
- `unknown` -> observe/probe before deletion

## Rule

Do not remove a surface just because it looks abstract. First ask whether it is a necessary feedback or control surface.
