# Closure handoff contract

Before invoking `verification-closure`, compile a closure handoff packet with:

- artifact state label and ID
- objective, scope, and constraints
- companion skill ledger
- routing and budget ledger
- warrant intake / parallelism plan
- truth units and invariant ledger
- findings, soundness, hazard, complexity, and negative-evidence ledgers
- adversarial action ledger and unresolved vetoes
- proof receipts tied to current artifact state
- one-change challenge result
- closure gate preview
- residual uncertainty

Closure must not proceed while material findings, soundness rows, review
counterexamples, unresolved adversarial vetoes, stale proof, or unretired additive
scaffolds remain open.
