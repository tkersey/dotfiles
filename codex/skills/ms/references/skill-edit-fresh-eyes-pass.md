# Skill-Edit Fresh-Eyes Pass for ms

Before the final report on any non-trivial skill creation or update, reread the skill as both a user and the router.

Check:
- Did the description trigger become too broad, too narrow, or duplicative of another skill?
- Did body workflow text conflict with frontmatter trigger guidance or companion-skill boundaries?
- Did the edit add workflow owned by `$refine`, `$tune`, `$seq`, `$ship`, `$land`, or another more specific skill?
- Did evidence or handoff requirements get weaker?
- Did frontmatter, path naming, resource links, or `agents/openai.yaml` drift?
- Would this change cause false activations, missed activations, or partial activations?

If anything material is found, fix it before the final report. If no material issue is found, report `fresh_eyes_delta: none`.
