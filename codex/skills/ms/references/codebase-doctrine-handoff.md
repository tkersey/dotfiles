# Codebase Doctrine Handoff

Use `$ms` only after `$codebase-doctrine` produces an accepted `CBSH-v1` and the user explicitly authorizes skill creation.

The handoff must bind:

```text
CBD doctrine and artifact-state IDs
candidate and proposed skill name
governing laws
triggers and non-triggers
consequential decisions
prohibited routes
required artifacts
success/failure signals
protected doctrine IDs
allowed package
validation
```

Do not create rejected or provisional candidates.

Do not turn the repository directory tree into skill packages.

Create the root repository skill first unless the user authorizes a focused skill independently.

When SKDC-v1 is appropriate, preserve doctrine law IDs in clause rationale or references without coupling runtime activation to the full CBD artifact.
