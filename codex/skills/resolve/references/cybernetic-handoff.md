# Cybernetic Handoff

Use `$cybernetic` when review or validation findings reveal a system behavior, not isolated defects.

Trigger when:

- same-cluster findings recur;
- fixes change incentives or feedback timing;
- review process itself is generating accumulation;
- local metrics conflict with whole-system outcome;
- the correct move may be feedback redesign, stop rule, distillation, or rule change.

Expected output:

```yaml
cybernetic_packet:
  system_type:
  stocks_and_flows:
  feedback_loops:
  leverage_map:
  intervention_design:
```
