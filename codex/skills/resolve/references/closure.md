# Intent-Closed Closure

## Tuple closure

A tuple may close when:

```text
current AC/horizon
no open batch
current CEB/MBK/RC/design/realization
targeted conformance clean
PHI strict-progress gate pass
current-head proof
PR threads swept
commit/push current
```

Tuple closure is not terminal when terminal holdout is pending or new review evidence can still enter the campaign.

## Terminal closure

Requires:

```text
one sealed terminal holdout
zero novel in-horizon CEX
zero unknown intent relation
zero contract-invalidating unresolved evidence
zero orphan code/proof
terminal PHI
all required laws proved
current delivery tuple
explicit reopen conditions
```

## Reopen conditions

```text
AC rebase
new in-horizon CEX
same-class recurrence
stale kernel/RC/design
artifact head change
proof invalidation
PR thread reopening
delivery mismatch
```

Never report unqualified `complete`.

Report exact tuple, horizon fingerprint, holdout receipt, and closure kind.
