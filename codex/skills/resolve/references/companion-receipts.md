# Companion Receipts

When ambiguous companions are explicitly invoked, leave compact receipts.

## Fixed point

```yaml
fixed_point_receipt:
  selected_route:
  owner:
  negative_route_gate:
    query_or_map:
    ledger_cli:
    store:
    exit_code:
  proof_required:
  handoff_result:
```

## Verification closure

```yaml
verification_closure_receipt:
  artifact_state:
  proof_current: yes | no
  closure_scope:
  unresolved_risk:
  closure_allowed:
```

## Cybernetic

```yaml
cybernetic_context:
  route_changed: yes | no
```

If no receipt exists, report root-equivalent behavior instead of claiming distinct skill contribution.
