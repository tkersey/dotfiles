# Skill-Edit Fresh-Eyes Pass for Refine

Before the final report on any non-trivial skill edit, reread the modified skill as both a user and the router.

Check:
- Did the trigger description become too broad or too narrow?
- Did the edit conflict with companion-skill boundaries?
- Did workflow text belong in another skill?
- Did evidence or handoff requirements get weaker?
- Did naming drift, stale paths, duplicate instructions, or frontmatter violations appear?
- Would this change cause false activations, missed activations, or partial activations?

If anything material is found, fix it before the final report. If no material issue is found, report `fresh_eyes_delta: none`.
