# universalist

`$universalist` is a boundary-triggered, proof-carrying architecture workflow. It preserves the full categorical corpus, but runs that knowledge as a hidden optimizer over a Universal Problem IR. Normal output is repository-native engineering language; mathematical derivations are opt-in.

The operating discipline is:

```text
one signal
one seam
one smallest honest construction
one law
one falsifier
```

## Decision flow

```text
repository facts
  -> smallest boring repository-native candidate
  -> task-local comparison universe
  -> hidden universal shadow
  -> material architectural delta gate
  -> effective lowering or obstruction
```

The shadow survives only when it changes ownership, representable states, legal composition, information flow, interpretation, construction paths, proof obligations, resources, existence, or migration. A categorical label without such a delta is discarded.

## Use

`$universalist` is implicitly invoked whenever implementation, refactoring, review, migration, or resolution considers how values, effects, state, evidence, authority, or behavior cross an owned code boundary. Activation does not imply refactoring: preserving an already exact seam is a valid result.

The skill starts with ordinary artifacts—records, tagged unions, checked wrappers, adapters, compatibility objects, state machines, explicit IRs, handlers, graphs, context parameters, and pointwise operations. It escalates only when the hidden theorem pass produces a stronger artifact and proof.

## Universal Problem compiler

The machine-readable theorem registry and split theorem-card directory contain ordinary universal constructions, Kan transport and lifts, Yoneda/Coyoneda, effects and coalgebras, Freyd geometry, operads and network composition, Day/promonoidal products, Tambara/contextual morphisms, density/codensity, categorical data/CQL, comonadic locality, and sheafification.

```bash
cd codex/skills/universalist
uv run --with pyyaml python3 scripts/compile_universal_problem.py \
  --problem path/to/problem.json \
  --format markdown
```

Default output hides expert names. Add `--explain-theory` for a mathematical audit.
