# Abstraction Replacement Elaboration

`kan` does not decide that an abstraction should be replaced. `universalist` does. `kan` elaborates the chosen replacement when the repair is categorical.

## Output checklist

1. Name the current abstraction and exactness gap.
2. State the sheaf failure.
3. Select the categorical construction.
4. Give data representation.
5. Give construction / eliminator / interpreter / projection.
6. Give law tests and falsifier.
7. Explain migration of one witness slice.
8. State what would refute the categorical framing.

## Common elaborations

- Optional-field soup -> coproduct/refined type.
- Repeated invariant -> equalizer/refined constructor.
- Shared-key agreement -> pullback witness.
- Callback API -> defunctionalized operation IR.
- Many interpreters -> free syntax / initial algebra.
- Contract-derived internals -> Kan lift / realizer / residual obligations.
- Projection sprawl -> Yoneda observation vocabulary.
- Generated/provenance drift -> Coyoneda payload + path.
- State/protocol confusion -> behavioral coalgebra.
- Raw context -> Context Compilation / Context Certificate.
