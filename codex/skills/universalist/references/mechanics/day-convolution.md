# Day Convolution and Promonoidal Description Composition

Use this reference only after Universalist has identified the seam, index world, base composition, indexed descriptions, witness slice, observations, law, falsifier, and effective implementation boundary.

Day convolution is not a synonym for pairing, merging, sequencing, numerical convolution, or operadic substitution.

## 1. Architectural purpose

Day convolution answers:

```text
Given a world whose indices compose,
how should descriptions indexed by that world compose?
```

It is the canonical colimit-preserving lift of a base tensor into a functor or presheaf category.

```text
base objects compose
  -> representable descriptions compose the same way
  -> arbitrary descriptions compose by distributing over colimits
```

Universalist reading:

```text
Index world owns admissible composition.
Description layer owns static/indexed artifacts.
Day convolution transports the index law to descriptions.
Interpreter/handler owns executable semantics.
```

## 2. Formal definition

Let:

```text
(C, tensor, I)  small monoidal category
(V, odot, 1)    cocomplete monoidal semantic category
F,G : C -> V    indexed descriptions
```

Assume `odot` preserves the required colimits separately in each variable.

The covariant Day convolution is:

```text
F star G = Lan_tensor(F external-product G)
```

where:

```text
(F external-product G)(a,b) = F(a) odot G(b)
```

Pointwise:

```text
(F star G)(c)
  ~= coend_{a,b}
       C(a tensor b, c) * (F(a) odot G(b))
```

For `Set`-valued functors, a representative has:

```text
a, b
x : F(a)
y : G(b)
k : a tensor b -> c
```

modulo the coend equivalence generated by coherent changes of `a` and `b`.

For presheaves `C^op -> V`, the variance commonly appears as:

```text
(F star G)(c)
  ~= coend_{a,b}
       C(c, a tensor b) * (F(a) odot G(b))
```

Always record the variance. Do not copy a formula from the opposite convention.

The unit is represented by the monoidal unit:

```text
J(c) = C(I, c)
```

or its presheaf-variance counterpart.

## 3. Programming interpretation

A direct existential representation has the shape:

```haskell
data Day f g a where
  Day :: f x -> g y -> (x -> y -> a) -> Day f g a
```

The hidden `x` and `y` are intermediate indices/results. The combining function witnesses how they produce `a`.

The general indexed form is richer:

```text
Description at c
  = choose a legal decomposition a tensor b -> c
  + description at a
  + description at b
  + quotient equivalent intermediate choices
```

The quotient is part of the architecture. It prevents accidental representation choices from becoming public semantic distinctions.

## 4. Completion property

Let `y` denote the Yoneda/representable embedding. Day convolution is chosen so that:

```text
y(a) star y(b) ~= y(a tensor b)
```

This is the key preservation law:

> Atomic descriptions compose exactly as the underlying atoms compose.

Arbitrary descriptions are colimits of representables, so their composition is determined by cocontinuous extension.

Software interpretation:

```text
primitive indexed cases determine composite indexed cases;
manual special cases should not redefine the composition law;
alternatives distribute through composition;
normalization removes presentation-only distinctions.
```

## 5. Monoids and lax monoidal interpreters

A monoid object under Day convolution consists of:

```text
multiply : F star F -> F
unit     : J -> F
```

By the Kan-extension universal property, this is equivalent to coherent maps:

```text
F(a) odot F(b) -> F(a tensor b)
1 -> F(I)
```

That is lax monoidal structure.

Universalist use:

```text
A description family is closed under composition
when its interpreter/composer is lax monoidal over the index world.
```

Examples:

- `Applicative`-like static computation descriptions;
- graded plans closed under grade composition;
- weighted languages closed under concatenation;
- resource predicates closed under splitting;
- species closed under label partition;
- spatial descriptions closed under product patches.

Do not claim a monoid merely because a binary function exists. Require unit, associativity, naturality/reindexing coherence, and observation-relative tests.

## 6. Closedness and residuals

When the Day tensor is closed/biclosed, there are residuals/internal homs:

```text
F star -  left-adjoint-to  F \ -
- star F  left-adjoint-to  - / F
```

Order-enriched reading:

```text
F star G <= H
  iff
G <= F \ H
```

Software reading:

```text
Given what is already supplied by F,
what additional description/resource/context G
would be sufficient to reach H?
```

Potential uses:

- residual capability requirements;
- missing resource assertions;
- context gaps;
- weakest additional plan fragment;
- protocol obligations;
- remaining evidence needed for a decision.

This is valid only when the description category has the relevant order/closed structure and the residual is effectively computable or approximated.

## 7. Promonoidal generalization

A monoidal tensor induces a composition kernel:

```text
P(a,b;c) = C(a tensor b, c)
```

A promonoidal category replaces this representable kernel with an arbitrary profunctor-like relation/object:

```text
P(a,b;c)
```

Convolution becomes:

```text
(F star G)(c)
  ~= coend_{a,b}
       P(a,b;c) odot F(a) odot G(b)
```

Use a promonoidal kernel when composition is:

- partial;
- relation-valued;
- nondeterministic;
- witnessed in several distinct ways;
- constrained by disjointness or capabilities;
- dependent on interface matches;
- provenance-sensitive.

Examples:

```text
Compose(heap1, heap2; heap)
  only when heap domains are disjoint.

Compose(effect1, effect2; effect)
  only when footprints are compatible.

Compose(rule1, rule2; composite)
  for each legal interface match.

Compose(context1, context2; requirement)
  for each lawful requirement decomposition.
```

Keep composition witnesses when provenance or explanation matters. Quotient only the distinctions declared presentation-only.

## 8. Description-product selector

| Pressure | Select | Reject nearby confusion |
| --- | --- | --- |
| Same-index independent combination | Pointwise/Hadamard | no decomposition sum |
| Every tensor decomposition contributes | Day convolution | not ordinary product |
| Partial/relation-valued decomposition | Promonoidal convolution | do not totalize invalid pairs |
| Recursive insertion into typed slots | Substitution/plethysm | not Day/Cauchy product |
| Value-dependent sequencing | Endofunctor composition/monad | not applicative/static Day |
| Ordered runtime effects | Freyd/premonoidal execution | description tensor does not grant interchange |
| Agreement through common observation | Pullback | not convolution |
| Gluing through explicit overlap | Pushout | not convolution |

Before selecting, name one counterexample that distinguishes the chosen product from the nearest alternative.

## 9. Established software patterns

### 9.1 Graded/Cauchy families

For natural-number grades with addition:

```text
(F star G)(n)
  = sum over i+j=n of F(i) odot G(j)
```

Use for:

- length/arity-indexed syntax;
- cost/resource grades;
- phase-indexed artifacts;
- bounded protocol steps;
- polynomial or power-series semantics.

Effective implementation:

- sparse arrays/maps;
- bounded convolution loops;
- dynamic programming;
- semiring aggregation;
- FFT/numerical methods only when the coefficient semantics genuinely support them.

### 9.2 Applicative/static effect descriptions

For endofunctors, the Day tensor under cartesian product supports the standard `Applicative` reading.

Use when the complete operation/dependency shape is known before intermediate results.

This enables:

- static dependency extraction;
- batching;
- documentation generation;
- cost analysis;
- validation before execution;
- test/production/explanation interpreters.

Select monadic/free-monadic syntax instead when later operation structure depends on earlier values.

Runtime order still belongs to Freyd/premonoidal or resource mechanics.

### 9.3 Resource predicates and separation

Let `R` have partial composition. For predicates `P,Q : R -> Bool`:

```text
(P star Q)(r)
  iff exists r1,r2.
       Compose(r1,r2;r)
       and P(r1)
       and Q(r2)
```

This is the shape of separating conjunction.

Use for:

- disjoint heaps;
- ownership regions;
- capabilities;
- noninterfering effect footprints;
- permission splitting;
- deployment/resource envelopes.

Falsifier: overlapping or otherwise incompatible resources are accepted without an admissibility witness.

### 9.4 Formal and weighted languages

With words and concatenation:

```text
(F star G)(w)
  = aggregate over w = uv of F(u) odot G(v)
```

Boolean aggregation yields language concatenation. Semiring aggregation yields weighted languages, costs, probabilities, multiplicities, or provenance.

Use finite automata, dynamic programming, or symbolic representations rather than enumerating every split when scale requires it.

### 9.5 Combinatorial species

For finite labels with disjoint union:

```text
(F star G)[S]
  = sum over S = A disjoint-union B
      F[A] x G[B]
```

Use when labels/names/resources split independently between structures.

Distinguish this Cauchy/Day product from species substitution, which recursively inserts structures into slots/blocks.

### 9.6 Operads and operation collections

Operation collections may carry a Day/Cauchy-like tensor, while operadic substitution supplies a different monoidal product under which operads are monoids.

Universalist distinction:

```text
Day convolution:
  combine indexed operation collections over decompositions.

Operadic substitution:
  insert composite operations into typed input slots.
```

Do not route typed recursive assembly to Day merely because arities are indexed.

### 9.7 Rule and graph-rewrite collections

DPO/SPO mechanics define individual rewrites. A convolution over matches/interfaces can compose collections of rules by ranging over legal matches.

Use only when:

- the rule-interface category/kernel is explicit;
- composition witnesses are retained;
- pushout-complement obstructions remain visible;
- the convolution does not replace the underlying rewrite laws.

### 9.8 Comonadic spatial convolution

For endofunctors on `Set`, Day convolution with respect to cartesian product lifts to suitable small comonads.

For density comonads generated by subbases `P1` and `P2`, the product is generated by external-product patches:

```text
<P1> star <P2>
  ~=
<(U,V) |-> P1(U) x P2(V)>
```

Universalist reading:

```text
Density generates locality.
Day convolution composes locality.
Sheafification glues meaning within locality.
```

Use for joint dependency/capability/evidence/ownership spatial worlds when the external-product patch system changes code or proof shape.

Require:

- combined point representation;
- product/subbasis construction;
- combined halo and labels;
- continuous projections;
- finite/indexed/incremental representation;
- a counterexample showing ordinary list concatenation is insufficient.

Do not assume the Day tensor of arbitrary comonadic spaces is the ordinary cartesian product of point sets.

### 9.9 Exact Context and residual obligations

Architecture inference only:

If requirements form a monoidal/promonoidal index world and context fragments are indexed by them, convolution can combine fragments over lawful requirement decompositions.

A residual may expose missing context:

```text
KnownContext star MissingContext <= RequiredContext
```

Do not use this framing for ordinary retrieval union, evidence deduplication, pullback joins, or pushout reconciliation without a real index composition.

## 10. Effective representations

A mathematical coend does not provide an efficient implementation automatically.

Record one implementation strategy:

### Finite/sparse support

```text
Map<Index, Payload>
```

Enumerate only nonempty index pairs.

### Bounded decompositions

Limit by grade, depth, resource budget, arity, time window, or rule-match policy. Record omitted legal cases as an approximation boundary.

### Dynamic programming

Memoize sub-decompositions. Use semiring aggregation when duplicates combine algebraically.

### Symbolic kernel

Represent `P(a,b;c)` as constraints, BDDs, SAT/SMT formulas, database queries, automata, or typed match objects.

### Canonical quotient

Use normalization, canonical IDs, union-find, or hash-consing only when the coend equivalence is precisely stated.

### Incremental convolution

Maintain dependency indexes and invalidate only outputs affected by changed description support or composition witnesses.

### Obstruction

Return an obstruction when:

- decomposition is infinite without a productive representation;
- quotient equality is undecidable for the required observations;
- collision loses semantic identity;
- resource cost exceeds the architecture budget;
- no suitable colimit/tensor preservation assumption holds.

## 11. Coend collisions and identity loss

The quotient may identify distinct representatives:

```text
(a,b,x,y,k)
~
(a',b',x',y',k')
```

This is intended when differences are only coherent reindexings. It is a defect when the difference carries required:

- provenance;
- ownership;
- effect order;
- local/global identity;
- security label;
- rule-match witness;
- cost attribution.

A certificate must state:

```text
Equivalence generated by:
Information intentionally forgotten:
Information retained outside the quotient:
Canonical representative:
Collision falsifier:
```

Do not assume convolution preserves monomorphisms or injective representations without evidence.

## 12. Law catalogue

### Representable law

```text
y(a) star y(b) ~= y(a tensor b)
```

Engineering witness: convolving two atomic descriptions normalizes to the same public behavior as composing their indices directly.

### Unit law

```text
J star F ~= F ~= F star J
```

### Associativity law

```text
(F star G) star H ~= F star (G star H)
```

### Soundness

Every composite carries a valid decomposition/admissibility witness.

### Completeness

Every supported legal decomposition contributes, or is named in the approximation boundary.

### Quotient coherence

Reindexing a representative does not change its normalized observable result.

### Interpretation/lax-monoidal law

```text
interpret(convolve(F,G))
  ==
combine(interpret(F), interpret(G))
```

### Residuation law

```text
F star G <= H iff G <= F \ H
```

when the relevant order/closed structure exists.

### Effect-order law

No reordering/parallelization without observational commutativity, centrality, or resource-disjointness evidence.

### Resource law

Decomposition enumeration, normalization, memory, cache invalidation, and failure are within the declared budget.

## 13. Falsifier catalogue

- base tensor/kernel cannot be named;
- description family is not actually indexed by the proposed world;
- pointwise product is mislabeled as Day;
- operadic substitution is mislabeled as Day;
- value-dependent monadic sequencing is forced into applicative structure;
- illegal partial composition is admitted;
- legal decomposition is omitted;
- quotient collapses semantic provenance or order;
- representable law fails;
- static structure is used to justify effect parallelism;
- decomposition complexity is unbounded without strategy;
- a numerical convolution implementation is being discussed without a functor-category architecture issue.

## 16. Compact selection rule

```text
Use Day convolution when:
  the unknown is composition of indexed descriptions,
  the index world already has lawful composition,
  all legal decompositions should contribute,
  coherent presentation choices should be quotiented,
  and the result has an effective implementation.

Use promonoidal convolution when:
  composition is partial, relational, or multi-witnessed.

Otherwise choose the smaller honest product.
```
