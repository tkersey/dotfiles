# Boundary Normal Form

A codebase is in Boundary Normal Form when every meaningful cross-world composition boundary has a Composition Certificate and all bypasses are removed or explicitly justified as primitive effects.

## Boundary Normal Form checklist

For each boundary:

- worlds are named;
- boundary kind is named;
- unknown location is identified;
- canonical artifact is chosen;
- interpreter/projection/lowering/handler is centralized;
- positive law witness exists;
- falsifier/negative witness exists;
- bypasses are removed, blocked, or recorded as primitive exceptions;
- status is verified, obstructed, planned, or primitive exception.

## Accretive workflow

Do not rewrite the codebase. Certify one seam at a time:

```text
uncertified glue
  -> boundary inventory
  -> Composition Certificate
  -> canonical artifact
  -> witness law
  -> bypass removal
  -> verified boundary
```

## Audit output

A Boundary Normal Form audit should produce:

- certified boundaries;
- uncertified boundaries;
- primitive exceptions;
- obstructed boundaries;
- next highest-leverage certificate;
- recommended law tests.

## Agentic coding

For coding agents, the ideal task is:

```text
Find one uncertified composition boundary.
Produce a Composition Certificate.
Implement the canonical artifact.
Add the witness law.
Remove one bypass.
Stop.
```
