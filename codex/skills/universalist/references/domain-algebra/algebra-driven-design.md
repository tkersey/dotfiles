# Domain Algebra Discovery

This is the Algebra-Driven Design kernel folded into Universalist as Track A0.

Use it to make a local world real before escalating to universal-architecture mechanics.

## Core shape

```text
Domain:
Carriers:
Operations / constructors / eliminators:
Observations / equality criteria:
Laws:
Non-laws / counterexamples:
Interpreters / effect boundaries:
Property tests / falsifiers:
Architecture implications:
Escalation candidates:
```

## Rule

```text
Algebra before architecture.
```

Do not choose `Lan`, `Ran`, lift, sheafification, exact context, effect handler, coalgebra, or operad until the local domain algebra has exposed its carriers, operations, observations, laws, and non-laws.

## Why it belongs inside Universalist

ADD discovers the algebra of one world. Universal Architecture governs boundaries between worlds.

```text
carriers      -> objects / inhabitants
operations    -> transformations / constructors / eliminators
observations  -> equality and semantic probes
laws          -> invariants and proof signals
non-laws      -> falsifiers and obstruction reports
interpreters  -> syntax/semantics boundaries
property tests -> executable witnesses
```

## Escalation

- If laws are local and executable, stay in Track A0/B.
- If laws fail across a boundary, use Track D or E.
- If observations determine context readiness, use Track F.
- If call sites disagree or glue non-uniquely, use Track G.
- If executable behavior is opaque, use Track H.
- If the whole computational substrate is in play, use Track I.
