# Universal Problem Intermediate Representation

## Purpose

Universalist keeps its full categorical theorem library, but category theory is an internal optimizer rather than the default user-facing vocabulary.

The root workflow is:

```text
repository facts
  -> one owned boundary
  -> boring repository-native candidate
  -> universal-problem IR
  -> hidden categorical shadow
  -> material-delta gate
  -> selected effective presentation or obstruction
  -> law, mediator, falsifier, and resource witness
```

The user should normally see types, schemas, constructors, adapters, interpreters, handlers, state machines, migration paths, tests, and obstructions. Expert categorical names are emitted only when explicitly requested or when a mathematical audit requires them.

## Governing rule

```text
Category theory must beat the boring architecture.
```

A categorical route is consequential only when it changes at least one material dimension:

```text
owner
representable states
legal composition
information flow
interpreter / projection ownership
public construction paths
proof obligations
resource behavior
existence versus obstruction
migration strategy
```

If the hidden shadow changes none of these, discard it and keep the boring candidate.

## Universal problem

For one seam, define the task-local class of admissible solutions.

```text
Objects:
  candidate repository-native realizations

Transformations:
  allowed behavior-preserving adapters, refinements, projections, migrations,
  interpreters, normalizations, or contextual transports

Observations:
  what downstream consumers are sanctioned to distinguish

Equivalence:
  when two candidates count as the same for this task

Constraints:
  compatibility, authority, provenance, effect order, state, and public contracts

Resources:
  time, space, latency, throughput, invalidation, capability, and deployment limits
```

Then identify the architectural hole:

```text
object
map
extension
realizer
composition
representation
observation surface
local-to-global glue
```

The shadow asks whether there is a distinguished effective completion among the admissible alternatives. The answer may be:

```text
exact universal solution
effective bounded approximation
ordinary candidate already exact
several incomparable lawful solutions
underdetermined specification
obstruction / no effective solution
```

## The hidden instruction set

The theorem registry compiles categorical knowledge into twelve internal instructions:

```text
DECLARE    define admissible artifacts and transformations
CONSTRAIN  satisfy simultaneous equations or agreement
GLUE       integrate contributors along declared overlap
GENERATE   create the least structure closed under operations
OBSERVE    characterize by sanctioned probes, traces, or views
TRANSPORT  move semantics coherently across a boundary
REALIZE    construct behind a fixed projection, or fail honestly
COMPOSE    make sequencing, wiring, decomposition, or feedback explicit
FRAME      preserve a capability under context extension
LOCALIZE   retain neighborhoods, restriction, labels, and provenance
NORMALIZE  quotient equivalent presentations safely
OBSTRUCT   prove why no honest or effective solution exists
```

Products, pullbacks, Kan extensions, Day convolution, Tambara modules, density comonads, sheafification, and the rest are theorem cards over this common protocol rather than independent top-level workflows.

## Universal witness contract

A positive local law is not enough for a universal claim. When the categorical shadow is consequential, record:

1. **Existence** — the artifact, map, or effective approximation can be constructed.
2. **Commutation** — required paths and observations agree.
3. **Mediation** — every admissible competing solution has the required comparison path.
4. **Canonicality** — that comparison is unique up to declared observation or normalization, or the approximation limit is explicit.
5. **Effectivity** — construction, comparison, normalization, and validation fit the resource model.
6. **Falsifier** — the nearest weaker or invalid route fails observably.

Engineering approximations may implement mediation and uniqueness through opaque constructors, canonical identifiers, normalized IRs, one public interpreter, or removal of competing bypasses. Do not claim literal universality when only a bounded approximation is implemented.

## Universal shadow protocol

### 1. Produce the boring candidate

Use repository conventions and the ordinary ladder first:

```text
record
tagged union
checked constructor
adapter
explicit context parameter
state machine
operation IR
handler
canonical merge
ordinary graph or query
```

### 2. Build the comparison universe

Declare:

```text
owner
source and target worlds
allowed transformations
required observations
equivalence / normalization
effects and ordering
authority and provenance
compatibility
resource budget
```

### 3. Run theorem cards

Use `references/universal-construction-registry.yaml`. A card is eligible only when its structural preconditions are evidenced. Missing preconditions narrow or block the route; they are never invented.

### 4. Compare material deltas

Keep the shadow only if it changes a material dimension. Otherwise set advanced mechanics to `none` and retain the boring candidate.

### 5. Lower to repository-native code

Separate semantic selection from presentation extraction:

```text
universal semantics
  -> valid presentation class
  -> repository conventions and cost model
  -> concrete type / schema / IR / handler / query / test
```

Category theory does not choose between equivalent language idioms unless the representation changes a law, observation, or resource obligation.

### 6. Erase the derivation

Default human-facing output uses plain engineering language. Preserve the internal construction key, evidence, and proof obligations in the plan/certificate. Emit the expert name only on explicit request.

## Obstruction discipline

Obstruction is a successful result when:

- a public observation boundary forgets implementation-critical evidence;
- several lawful realizations remain incomparable and no selection policy exists;
- a claimed composition lacks its tensor, action, overlap, or observation structure;
- a required quotient collapses observable provenance or order;
- a free, cofree, dense, all-context, or decomposition construction has no effective presentation;
- authority, effect order, or resource ownership is underdetermined.

Do not synthesize defaults merely because an abstract specification composes lawfully.

## Anti-overreach

- A categorical name without a material delta is discarded.
- A commuting square without competitor mediation is only a local law.
- A wrapper is not a context action.
- A nested loop is not a decomposition theorem.
- A dependency graph is not automatically a coherent locality model.
- Example coverage is not canonical reconstruction.
- Static composition does not prove runtime commutativity.
- A generalized relation is not automatically representable by a function.
- A mathematically existing construction is not an architecture until it has an effective presentation and resource witness.
