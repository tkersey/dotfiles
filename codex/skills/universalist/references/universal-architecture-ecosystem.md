# Universal Architecture Ecosystem

Universal architecture designs software around canonical boundary artifacts, compositional change calculi, lawful indexed-description composition, lawful context actions, and effective locality:

```text
free syntax
coherent observations
transported semantics
lifted implementations
free builders and obstructions
behavioral coalgebras and effect handlers
Freyd/premonoidal effect geometry
operadic composition grammars
double categories, equipments, compatibility squares, and interchange
pointwise, Day, and promonoidal description products
Tambara modules and optic/residual morphisms
comonadic spaces, bases, halos, germs, and continuous boundaries
explicit IRs and law tests
```

Ordinary products, sums, checked constructors, adapters, one compatibility witness, labelled graphs, context objects, profunctors, residual records, queries, and state machines remain first choices. Escalate only when a stronger structure changes architecture or proof.

## Operating rule

Ask:

1. Is an ordinary repository-native construction enough?
2. Is the pressure boundary shape, two-dimensional process/change composition, indexed-description composition, context framing, locality, transport, or proof?
3. For double categories, can two arrow families, both compositions, typed squares, both pastings, interchange/coherence, interpretation, and resources be named?
4. For equipment, are companions, conjoints, or restrictions actually realizable?
5. Can one witness seam and law test make the artifact honest?
6. Would the structure prevent mismatched squares, noncompositional migrations, drift, invalid states, illegal composition, lost provenance, or invented correctness?

If not, keep the ordinary architecture or return unresolved/obstruction.
