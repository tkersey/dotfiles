# Abstraction Manipulator Playbook

## Purpose

Use this playbook when an existing abstraction feels wrong but it is not obvious what should replace it.

## Steps

1. Inventory current abstraction and semantic load.
2. Identify local usage contexts.
3. Extract local meanings as sections.
4. Identify overlaps.
5. Check overlap compatibility.
6. Check whether compatible locals have a global representation.
7. Check whether global representations are unique.
8. Classify the possibility-envelope gap.
9. Select canonical repair.
10. Produce a Sheafification Certificate.
11. Implement one witness slice.
12. Add gluing law and falsifier.
13. Remove bypasses.

## Construction selector

- Independent fields -> product.
- Exclusive alternatives -> coproduct.
- Stable predicate -> refined type / equalizer.
- Shared projection agreement -> pullback witness.
- Many interpreters over commands -> free syntax / initial algebra.
- Old semantics moves to new target -> transported semantics / Lan.
- New internals satisfy old views -> coherent observations / Ran / Yoneda.
- Public behavior determines internals -> Kan lift / realizer.
- Effects need multiple runtimes -> effect signature + handlers.
- Ongoing behavior over time -> behavioral coalgebra.
- Generated data loses provenance -> Coyoneda payload + path.
- Hidden cross-boundary behavior -> defunctionalized IR.
- Semantic consumer sees raw data -> Exact Context / Context Certificate.
