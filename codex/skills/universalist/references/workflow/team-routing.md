# Universalist Team Routing

Use the smallest roster that covers independent uncertainties.

## New system / major capability

- world cartographer;
- substrate architect;
- categorical architect;
- semanticist;
- resource realist;
- proof auditor after synthesis.

## Existing-code structural refactor

- world cartographer;
- categorical architect;
- semanticist when hidden behavior/effects exist;
- proof auditor;
- one implementer and verifier after selection.

## API or public-contract boundary

- world cartographer;
- categorical architect;
- semanticist;
- proof auditor.

## Performance, concurrency, distributed systems

- substrate architect;
- resource realist;
- semanticist;
- proof auditor.

## Exact context / data integration

- world cartographer;
- semanticist;
- categorical architect;
- resource realist when freshness/serving matters;
- proof auditor.

## Small bounded seam

Do not spawn a team. Use the root agent or one read-only specialist.
