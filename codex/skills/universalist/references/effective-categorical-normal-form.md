# Effective Categorical Normal Form

A system is in Effective Categorical Normal Form when category theory defines architecture-level composition while every construction is executable and operationally honest.

## Conditions

- worlds and boundaries are explicit;
- canonical artifacts own nontrivial composition;
- syntax and semantics are separated when behavior must be inspected;
- effects, state, and interaction are explicit;
- categorical constructions have finite/effective presentations;
- concrete primitives are registered and handled;
- observations define semantic equivalence;
- laws and falsifiers are executable;
- resource constraints are modeled;
- unsupported exact constructions are marked obstructed or approximated;
- one witness slice proves end-to-end realizability.

## Anti-normal forms

- theorem-shaped architecture with no algorithm;
- callbacks hiding an operation language;
- effects hidden in generic services;
- public behavior with no projection tests;
- recursion/nontermination left implicit;
- runtime performance treated as an implementation afterthought;
- external APIs described abstractly but not handled concretely;
- several writers implementing competing interpretations.
