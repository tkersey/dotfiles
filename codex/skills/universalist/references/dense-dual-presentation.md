# Dense-Dual Presentation

## Principle

When a semantic artifact is too large, infinitary, observational, or completion-like to present by generators and equations, try to present it by a small dense world of probes plus a duality or observation bridge.

```text
small probe world + density + dual/observation bridge
  -> semantic reconstruction / codensity-style presentation
```

## Architecture reading

Do not always build the big semantic object directly. Find the small probe world that densely presents it, then certify reconstruction by laws.

## Data to recover

- Target behavior / semantic artifact.
- Failed or awkward algebraic presentation.
- Small probe world: finite cases, legacy observations, trace probes, expectations, capability probes, policy decisions, test scenarios, or finite approximants.
- Dense map / coverage claim.
- Dualizing or observation object: Bool, JSON, Trace, Score, Distribution, PolicyDecision, Expectation, etc.
- Reconstruction operation.
- Domain-specific theorem or assumption.
- Reconstruction law and falsifier.

## Software examples

- Compatibility facade determined by old client probes.
- Agent competence determined by trace, policy, and tool-result probes.
- Probabilistic behavior determined by expectations or finite stochastic tests.
- Distributed protocol behavior determined by finite trace projections and safety observations.
- Semantic cache behavior determined by query observations.

## Guardrail

A dense-dual presentation is an architecture inference unless the dense functor, duality, and reconstruction theorem are modeled precisely. In ordinary software, use probe coverage tests and explicit domain assumptions.
