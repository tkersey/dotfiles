# Universalist

Use this skill whenever a code boundary is considered. The skill is a boundary-centered architecture compiler whose hidden optimizer is category theory.

Default discipline:

```text
one signal
one seam
one boring candidate
one universal shadow pass
one smallest honest construction
one executable proof
```

The category-theory derivation is normally internal. Emit repository-native types, schemas, constructors, interpreters, handlers, migrations, state machines, tests, or obstruction reports. Expose the expert categorical explanation only when the user requests it or when a review must audit the mathematical claim.

## Boundary-trigger mandate

Use this skill whenever implementation, refactoring, review, migration, or resolution considers a code boundary. Boundary consideration itself is the activation signal. A boundary is any owned seam where values, effects, state, evidence, authority, identity, context, or observable behavior cross between representations or responsibilities.

Apply the lens in both directions:

- **Implementation** — name the boundary and owner; state preserved, forgotten, generated, and observed information; name compatibility, law, and falsifier before mutation.
- **Resolution** — decide whether a finding is a boundary liability; repair it at the owning seam; reject symptom-only downstream patches; verify the disposition and law.

Activation is broad; escalation is narrow. An already exact boundary may be preserved. Do not invent categorical structure merely because the skill activated.

Do not trigger for work wholly internal to one unchanged boundary, pure formatting or UI polish, numerical convolution, or category-theory exposition with no concrete software seam.

## Trigger-to-evidence kernel

Record this receipt immediately:

```text
Boundary:
Disposition: preserved / introduced / changed / repaired / removed / bypass-justified
Owner:
Source / target:
Preserved / forgotten / generated / observed:
Compatibility:
Law:
Falsifier:
```

Then decide whether the route is consequential: did the boundary pass select or preserve one construction by rejecting a nearby alternative for a changed seam?

- If no, retain the compact receipt and continue the repository's ordinary workflow.
- If yes, allocate one ledger-addressed plan, emit exactly one root-scoped decision receipt, and complete the universal-problem gate before mutating the seam.

This remains an inner lens. It does not replace the repository's implementation, review, verification, or closeout workflow.

# Universal Problem Compiler

The compiler's hidden question is:

> Among the effective architectural solutions satisfying this seam's observations, equations, compatibility, authority, effects, and resource limits, is there a distinguished universal solution?

The answer may be:

```text
preserved
ordinary
exact universal completion
effective approximation
underdetermined
obstructed
primitive exception
```

## Phase 0 — Inspect reality

Before abstraction, inspect:

- current language and type features;
- framework and repository conventions;
- public, wire, storage, process, plugin, context, and runtime boundaries;
- existing construction paths and bypasses;
- sanctioned observations and compatibility fixtures;
- effect order, authority, failure, persistence, and deployment constraints;
- current test runner and fastest credible proof signal.

## Phase 1 — Produce the boring candidate

First produce the smallest ordinary repository-native candidate without advanced categorical vocabulary. Typical candidates are:

```text
record / struct
tagged union / enum with payload
checked constructor / value object
validated pair or join object
canonical merge with conflict policy
adapter / decoder / migration
strategy / function seam
AST or explicit IR
state transition plus observer
operation signature plus handlers
plain context parameter
labelled graph or bounded dependency index
```

The universal optimizer must beat this candidate. Category theory is not allowed to win by renaming it.

## Phase 2 — Construct the task-local comparison universe

Define the architecture policy before selecting a construction:

```text
Objects:
Admissible transformations:
Sanctioned observations:
Observational equivalence / normalization:
Required equations and compatibility:
Information allowed to be forgotten or generated:
Authority and capability policy:
Effect order / commutation / duplication / discard policy:
Resource budget:
Architectural hole:
```

The admissible transformations are often the most consequential choice. State what changes count as preserving the contract. A universal construction in the wrong comparison universe is the wrong architecture with a proof.

If this universe cannot be specified well enough to generate a law and falsifier, stay ordinary or report underdetermination.

## Phase 3 — Classify the hole

Use one primary hole kind:

```text
object          a missing owned representation
map             a missing adapter, interpreter, projection, or migration
composition     missing rules for legal assembly or sequencing
representation  a semantic artifact lacks an effective presentation
equivalence     duplicate presentations need a safe normal form
context         a capability must survive legal framing
locality        values need coherent neighborhoods and restrictions
proof           the artifact exists but lacks a witness or falsifier
```

And one polarity:

```text
receive     alternatives should map into the solution
emit        the solution should map to compatible consumers
extend      behavior must move along a boundary
realize     public observations must be implemented behind a projection
observe     sanctioned observations should determine the artifact
compose     legal combinations must be generated and interpreted
frame       capability must survive context extension
localize    neighborhoods and restrictions must be preserved
glue        compatible local or overlapping meanings need a global artifact
```

## Phase 4 — Select universal bytecode

Use the smallest sequence of internal operations:

```text
DECLARE     define worlds, maps, observations, equivalence, effects, resources
CONSTRAIN   retain exactly the simultaneous agreements or equations
GLUE        integrate contributors while identifying only declared overlap
GENERATE    create the least structure closed under declared operations
OBSERVE     characterize behavior through sanctioned probes or traces
TRANSPORT   extend, restrict, migrate, or lower semantics coherently
REALIZE     construct an implementation behind a fixed projection
COMPOSE     make legal sequencing, wiring, decomposition, or feedback explicit
FRAME       preserve a generalized capability under context action
LOCALIZE    represent coherent neighborhoods, restrictions, and provenance
NORMALIZE   quotient equivalent presentations without losing observations
OBSTRUCT    prove that no exact, canonical, representable, or effective solution follows
```

Do not select a theorem by name first. Select bytecode from the architectural pressure, then load the corresponding card from `references/universal-construction-registry.yaml`.

## Phase 5 — Instantiate a theorem card

Each theorem card supplies:

```text
preconditions
admissible competitors
mediator direction
existence obligation
commuting/preservation obligation
factorization/mediation obligation
canonicality or uniqueness-up-to obligation
effective repository-native lowerings
positive witnesses
falsifiers
resource hazards
boring fallback
expert name and references
```

Use `scripts/emit_universal_problem.py <concept-id>` for a scaffold. Use `--expert` only when the derivation should be surfaced.

A named construction is justified only when its card's preconditions are witnessed and its universal property changes code, tests, ownership, representable states, legal composition, or obstruction behavior.

## Phase 6 — Universal proof contract

A consequential universal claim requires all six dimensions:

1. **Existence** — the artifact, map, or effective approximation can be constructed.
2. **Commutation** — required paths, equations, and observations agree.
3. **Mediation** — every admissible competing solution can factor through the selected owner or artifact in the declared direction.
4. **Canonicality** — the mediator is unique up to the declared observational equivalence or normal form; engineering approximations use opaque construction, canonical IDs, or one public path.
5. **Effectivity** — construction, interpretation, mediation, normalization, and validation fit the time, space, latency, invalidation, capability, and deployment budget.
6. **Falsifier** — one nearby weaker, illegal, incomplete, nonrepresentable, or noneffective alternative fails observably.

A commuting square alone is not a universal proof. A passing happy-path fixture alone is not a universal proof.

## Phase 7 — Materiality gate

Compare the universal candidate with the boring candidate. Retain the categorical result only when it changes at least one of:

```text
boundary ownership
representable values, programs, states, or morphisms
legal composition or wiring
information preservation, forgetting, provenance, or quotienting
authority or capability transfer
interpreter, projection, or handler ownership
construction-path uniqueness or bypass policy
law, factorization witness, or falsifier
resource, invalidation, or deployment behavior
existence versus underdetermination versus obstruction
migration or compatibility strategy
```

If no material delta exists, select the boring candidate and record the theorem card as explanatory only or `none`.

This gate is mandatory. Category theory must beat the boring architecture.

## Phase 8 — Lower and erase

Choose the simplest effective presentation supported by the repository:

```text
native ADT / sealed hierarchy / tagged struct
private constructor / parser / validated factory
canonical schema / constrained view / query
AST / command IR / workflow syntax
interpreter / compiler / handler / renderer
state transition / observer / trace fixture
sparse indexed family / admissibility kernel / normalizer
residual IR / frame API / generated wrapper
labelled graph / bounded halo / incremental index
published context snapshot / certificate
obstruction type and diagnostic
```

Emit plain engineering language by default. Preserve the expert derivation in the plan or certificate, not in every user-facing response.

## Comonadic Spatiality Selector

Prefer an ordinary labelled graph or context object when it already makes the seam exact. Escalate only when locality changes correctness and center, nested-neighborhood, restriction, local/global identity, continuity, basis/reconstruction, and resource laws change the artifact or tests. # Track compatibility

Tracks remain reporting and workflow compatibility aliases; they are not the theorem-selection engine.

- **Track A0 — Domain Algebra Discovery:** carriers, operations, observations, laws, non-laws, interpreters, property tests, index/context/locality structure.
- **Track A — Diagnosis:** run the boring candidate and universal shadow; report the material delta.
- **Track B — One-seam refactor:** implement one selected lowering and stop after verification.
- **Track C — Staged migration:** preserve wire/storage/public shape behind adapters while strengthening the internal owner.
- **Track D — Canonical boundary:** a theorem card materially changes artifact, ownership, composition, or proof.
- **Track E — Composition certification:** produce a Composition Certificate and Boundary Normal Form disposition.
- **Track F — Exact Context:** prepare and certify a semantic-consumption boundary.
- **Track G — Possibility Sheafification:** repair local inconsistency, missing glue, non-unique glue, or hidden excess possibility.
- **Track H — Category Pivot / Easy-World Transfer:** move the hard operation into syntax, schemas, relations, coalgebras, resources, presheaves, or context-action structure, then interpret back.
- **Track I — Effective Universal Software Synthesis:** design a whole capability on an effective substrate with concrete primitives, effects, state, observations, resources, laws, and one witness seam.

# Decision observability and ledger-addressed plans

For Track B, C, D, F, G, H, or I—and for any consequential `UNI-PRESERVE` decision—create one fresh plan through:

```bash
scripts/init_universalist_plan.sh [PROJECT_ROOT]
```

Equivalent native form:

```bash
ledger create --source universalist \
  --repo PROJECT_ROOT \
  --template /path/to/universalist/templates/universalist-plan.md
```

Plans live at:

```text
.ledger/universalist/plan-{plan-id}.md
```

The plan id is `YYYYMMDDTHHMMSSnnnnnnnnnZ-NNNN`. Every create invocation must never reuse, truncate, or overwrite an earlier plan. Retain the exact plan id. Resolve it with:

```bash
ledger path --source universalist --repo PROJECT_ROOT --id PLAN_ID
```

Use `ledger latest --source universalist` only for recovery and verify the plan's task identity before resuming.

Use `references/decision-contract.yaml` as route authority. Before mutating a consequential seam, emit exactly one root-scoped receipt:

```bash
ledger emit --source universalist \
  --plan "$UNIVERSALIST_PLAN" \
  --contract /path/to/universalist/references/decision-contract.yaml \
  --question "Which construction owns this seam?" \
  --selected-route UNI-ORDINARY \
  --rejected-route UNI-CANONICAL \
  --expected-outcome "The owner boundary enforces one observable law." \
  --disposition changed \
  --construction "checked adapter at the owner boundary" \
  --law "required observations are preserved" \
  --falsifier "a mismatched source is accepted" \
  --advanced-mechanics none \
  --evidence-ref "code:path" \
  --write-plan
```

One plan represents one changed seam and admits one receipt. A distinct seam requires a distinct plan. Workers reference the root decision id and do not duplicate the receipt.

Record the ordinary candidate, universal problem, theorem card, mediation/factorization, canonicality, effectivity, and material delta in the plan. Keep machine receipts in the plan or trace; summarize the decision for humans rather than pasting raw telemetry unless the platform explicitly requires it.

# Team mode

Subagents are explicitly gated:

```text
Only spawn the team when the current user prompt asks for subagents,
parallel agents, team mode, the categorical substrate team, or equivalent delegation.
```

When authorized:

```text
2-6 read-only specialists map facts and independent uncertainties
root produces boring candidate and universal problem
categorical architect proposes theorem-card completion
semanticist and resource realist test observations/effects/effectivity
root synthesizes one candidate and one strongest countercase
proof auditor attacks existence, mediation, canonicality, and obstruction
one writer implements one witness seam
independent verifier checks the diff and proof
root integrates and stops
```

The root owns scope, the comparison universe, synthesis, write ordering, final decisions, and user-facing conclusions. Specialist packets are evidence, not truth. Only one writer mutates a seam. Child agents do not recursively delegate.

# Output contract

Honor the user's and repository's output format first. Embed the Universalist result rather than replacing the host workflow.

For non-trivial work include, in compact form:

1. **Track / disposition**
2. **Signal and seam**

Do not expose category-theory vocabulary merely to prove the optimizer ran. When expert explanation is requested, include the comparison universe, diagram, universal property, hypotheses, effective presentation, and claim boundary.

# Guardrails

- Algebra before architecture.
- One signal, one seam, one smallest honest construction.
- Category theory must beat the boring candidate.
- No universal claim without admissible competitors and a mediator.
- No canonicality claim without uniqueness up to declared observations or normal form.
- No categorical artifact without an effective presentation, interpreter/projection/handler, law, falsifier, and resource model.
- No external behavior without named concrete primitives and handlers.
- No semantic-equivalence claim without sanctioned observations.
- No context framing without a real context action, profunctor, coherence laws, and representability status.
- No Day convolution without a real shared index world or explicit product index, tensor/kernel, unit, decomposition witnesses, quotient, interpreter, and effectivity bound.
- No comonadic spatiality without center, neighborhood coherence, restriction, effective halo/basis, continuity, and a locality falsifier.
- No sheafification without a usage site, local sections, overlaps, gluing existence, uniqueness, and excess-state analysis.
- Do not infer effect commutativity, parallelism, duplication, or discard from applicative, Day, Tambara, optic, or static structure.
- Prefer repository-native types and tests over generic categorical libraries.
- Keep public, wire, and storage shapes stable behind adapters unless breaking change is requested.
- Stop after the first verified seam unless the user asked for a sweep.
- Return an obstruction rather than inventing missing evidence, authority, policies, mappings, bases, or resource bounds.
